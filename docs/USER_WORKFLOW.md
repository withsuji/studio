# 사용자 워크플로우 (User Workflow)

FocusFlow 애플리케이션의 핵심 사용자 시나리오를 정의합니다.

## 1. 프리랜서: 업무 시간 추적 및 청구 관리

### 시나리오 개요
프리랜서 사용자가 대시보드에 접속하여 캘린더 일정을 확인하고, 업무 시간을 청구 가능한 항목으로 변환하여 인보이스를 생성하는 과정입니다.

### 단계별 흐름
1.  **대시보드 진입 (`/dashboard`)**
    *   사용자는 로그인 후 대시보드 메인 화면에서 주요 지표(예상 수익, 집중 시간, 미처리 인보이스 등)를 한눈에 확인합니다.
2.  **일정 확인 및 시간 기록 (`/dashboard/calendar`)**
    *   'Calendar' 메뉴로 이동하여 주간 일정을 확인합니다.
    *   완료된 미팅이나 집중 업무 시간(Focus Block)을 확인합니다.
3.  **청구 가능 여부 설정 (`/dashboard/time-entries`)**
    *   'Time Entries' 메뉴로 이동하여 캘린더에서 동기화된 시간 기록 목록을 확인합니다.
    *   각 항목에 대해 **Billable Toggle**을 사용하여 청구 가능 여부를 설정합니다. (AI 자동 감지 기능 포함)
4.  **인보이스 확인 (`/dashboard/invoices`)**
    *   'Invoices' 메뉴에서 청구할 항목들이 포함된 인보이스 상태를 확인하고 관리합니다.

## 2. 외부 클라이언트: 미팅 예약

### 시나리오 개요
외부 클라이언트가 프리랜서의 예약 페이지 링크를 통해 가능한 시간에 미팅을 예약하는 과정입니다.

### 단계별 흐름
1.  **예약 페이지 접속 (`/book/[user]`)**
    *   클라이언트는 프리랜서가 공유한 예약 링크(예: `/book/alex-doe`)로 접속합니다.
    *   프리랜서의 프로필, 예약 가능한 시간대, 미팅 유형(예: 30분 미팅) 정보를 확인합니다.
2.  **날짜 및 시간 선택**
    *   캘린더 UI에서 예약 가능한 날짜를 선택합니다.
    *   해당 날짜의 가능한 시간 슬롯 목록에서 원하는 시간을 선택합니다.
3.  **예약 정보 입력**
    *   이름, 이메일, 미팅 관련 메모를 입력하는 폼이 나타납니다.
    *   정보를 입력하고 'Book' 버튼을 클릭합니다.
4.  **예약 완료**
    *   예약 성공 메시지와 함께 확정된 일시를 확인합니다.
    *   (시스템 내부 동작: 프리랜서의 캘린더에 일정이 추가되고, 이메일 초대장이 발송됩니다.)
